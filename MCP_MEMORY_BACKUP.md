## ns::pisignage::architecture

### Display Stack Architecture

```
Traditional (Pre-Trixie):
  X11 → VLC fullscreen

New Trixie Stack:
  greetd (init/session manager)
    ↓
  labwc (Wayland compositor)
    ↓
  Chromium (kiosk mode browser)

Legacy Support:
  VLC player remains fully functional
  X11 packages still installed
  All existing APIs unchanged
```

### Component Responsibilities

| Component | Role | Config Location |
|-----------|------|-----------------|
| greetd | Auto-login, session init | `/etc/greetd/config.toml` |
| labwc | Wayland compositor | `~/.config/labwc/rc.xml` |
| Chromium | Kiosk browser | Via autostart |
| kiosk-apply | Config generator | `/opt/pisignage/scripts/kiosk-apply` |
| API | Remote control | `/opt/pisignage/web/api/kiosk.php` |

### Data Flow

1. User/API updates config files in `/opt/pisignage/config/`
2. `kiosk-apply` script reads configs
3. Script generates `~/.config/labwc/autostart`
4. On next labwc session start, Chromium launches with specified URL/flags

## ns::pisignage::notes

### Design Decisions

1. **Wayland-First Approach**
   - No X11 tools used (no xdotool, unclutter, etc.)
   - Modern compositor (labwc) chosen for lightweight footprint
   - Future-proof for Raspberry Pi OS evolution

2. **POSIX Shell Scripts**
   - `kiosk-apply` uses POSIX sh (not bash)
   - Maximum portability
   - Minimal dependencies

3. **Feature Flag Pattern**
   - `ENABLE_KIOSK=0` allows instant rollback
   - No need to uninstall packages
   - Preserves VLC functionality

4. **Idempotent Design**
   - Re-running `kiosk-apply` is safe
   - Re-running `install.sh` is safe
   - Configs created only if missing

5. **Network Resilience**
   - 20-second network wait in kiosk-apply
   - Prevents race conditions on boot
   - Logged clearly for debugging

### Challenges & Solutions

**Challenge 1:** Need to support both Trixie and older OS versions
**Solution:** Detection in `install.sh` with `detect_os_version()` function. Trixie packages only installed when detected.

**Challenge 2:** Chromium restart requires labwc session restart
**Solution:** API restart endpoint kills Chromium; documented that full effect requires logout/login or labwc restart.

**Challenge 3:** Testing without physical Pi hardware
**Solution:** Mock environment tests in `smoke.sh` that work on any POSIX system.

**Challenge 4:** Avoiding shell injection in API
**Solution:** Input validation in `kiosk.php` (URL format check, regex for shell metacharacters).

### Known Limitations

1. **Immediate Chromium restart** requires labwc session restart (documented workaround)
2. **greetd auto-login** not configured by install.sh (manual step in UPGRADE_TRIXIE.md)
3. **Multi-monitor** requires manual `wlr-randr` configuration (documented in advanced section)

### Recommended Chromium Flags

**Minimal (default):**
```
--incognito --noerrdialogs --disable-translate --no-first-run
```

**Enhanced:**
```
--incognito --noerrdialogs --disable-translate --no-first-run --disable-infobars --disable-session-crashed-bubble
```

**4K Displays:**
```
--incognito --noerrdialogs --force-device-scale-factor=1.5 --high-dpi-support=1
```
